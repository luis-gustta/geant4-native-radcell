# Working With RADCELL/VascularTumor Experiments

This document explains how to make controlled local modifications to the original RADCELL `VascularTumor` workflow after the compatibility layer has been installed.

This repository does not define a high-level experiment-description format. It is intentionally limited to native Geant4 installation, RADCELL compatibility patching, build support, launcher creation, and first-run validation. Higher-level experiment abstractions belong outside this repository.

## Principle

Keep three things separate:

```text
patched RADCELL checkout        -> stable source tree used for compatibility work
external working directory      -> simulation outputs and logs
local experiment copy           -> controlled modifications of VascularTumor
```

Do not use the root of `geant4-native-radcell` as a simulation workspace.

Prefer:

```bash
mkdir -p ~/Desktop/radcell_workspace
cd ~/Desktop/radcell_workspace
```

## Create a local experiment copy

Start from a patched and working RADCELL checkout:

```bash
mkdir -p ~/Desktop/radcell_workspace/experiments

cp -a ~/radcell/RADCELL/VascularTumor \
  ~/Desktop/radcell_workspace/experiments/VascularTumor_short_test
```

Work only inside the copy:

```bash
cd ~/Desktop/radcell_workspace/experiments/VascularTumor_short_test
```

## Main files

| File | Role |
|---|---|
| `VascularTumor.cc3d` | CompuCell3D project entry point. |
| `Simulation/VascularTumor.xml` | Lattice size, plugins, fields, cell types, constraints, and simulation step count. |
| `Simulation/VascularTumor.py` | Python entry point for CompuCell3D. Usually should remain close to the patched version. |
| `Simulation/VascularTumorSteppables.py` | Cell dynamics, growth, death, division, radiation schedule, and calls into radiation transport. |
| `Simulation/RadiationTransportModule.py` | Calls `RADCellSimulation.py` using the current Python interpreter and patched runtime assumptions. |
| `Simulation/RADCellSimulation.py` | Prepares the RADCELL/Geant4 backend call and handles RADCELL-side input/output. |
| `Simulation/testInputSource.in` | Geant4/RADCELL source macro: particle, energy, position, direction, and event count. |
| `Simulation/cellInformation.csv` | Cell information file used by RADCELL. It may be generated by the workflow or used directly in backend tests. |

## Change the number of CompuCell3D steps

Inspect the XML:

```bash
grep -n "Steps\|MCS" Simulation/VascularTumor.xml
```

Edit the step count in `Simulation/VascularTumor.xml` for short tests or longer runs.

Use small values for smoke tests. Preserve the original value for reproducibility comparisons.

## Change when radiation is called

Inspect the steppables:

```bash
grep -n "mcs\|Radiation\|runMode\|simulationID" Simulation/VascularTumorSteppables.py
```

The original workflow may call radiation only at high MCS values. For a short test, make a local backup:

```bash
cd Simulation
cp VascularTumorSteppables.py VascularTumorSteppables.py.before_short_test
```

Then modify the radiation trigger locally, for example to test a call at early MCS.

After the test, either keep the modified copy as a named experiment or restore:

```bash
mv VascularTumorSteppables.py.before_short_test VascularTumorSteppables.py
```

## Change the radiation source

The typical input source file is:

```text
Simulation/testInputSource.in
```

Inspect relevant lines:

```bash
grep -n "particle\|ene\|energy\|beamOn\|pos\|ang" Simulation/testInputSource.in
```

Common changes include:

- particle type;
- monoenergetic energy value;
- number of events in `beamOn`;
- source position;
- source direction;
- source geometry if supported by the macro and backend.

After changing the source, test the backend directly:

```bash
cd ~/Desktop/radcell_workspace/experiments/VascularTumor_short_test/Simulation

~/run_radcell_cc3d_python.sh RADCellSimulation.py \
  "out test_source_change" \
  testInputSource
```

## Run a modified copy headlessly

The generic command is:

```bash
cd ~/Desktop/radcell_workspace

~/run_radcell_cc3d_python.sh -m cc3d.run_script \
  -i ~/Desktop/radcell_workspace/experiments/VascularTumor_short_test/VascularTumor.cc3d \
  --output-dir "$PWD/radcell_runs/VascularTumor_short_test"
```

Use the specialized helper script only when running the `VascularTumor` workflow from a standard patched RADCELL checkout.

## Rules for this repository

- Do not commit generated simulation directories.
- Do not commit large logs.
- Do not commit local RADCELL source-code copies.
- Do not commit modified RADCELL files directly unless they are represented as patches and are legally safe to distribute.
- Keep compatibility tooling separate from higher-level experiment interfaces.
